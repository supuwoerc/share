# 从河狸到伐木机器

> Q1 不同的伐木方法到底哪种最好？最可靠？<sup>1</sup>

<div class="mt-3 mb-3">
  <ul class="text-3">
    <li v-click><q>速度</q>：伐木机器 > 人工 > 河狸</li>
    <li v-click><q>成本</q>：伐木机器 > 人工 > 河狸</li>
    <li v-click><q>单位成本</q>：河狸 > 人工 > 伐木机器</li>
    <li v-click><q>可靠性</q>：河狸 > 人工 > 伐木机器</li>
  </ul>
</div>

<p v-click class="text-4">奥卡姆剃刀（Occam's Razor）：一个哲学和科学原则，主张在面对多种可能解释时，应优先选择假设最少、最简单的解释，因为它通常更接近真相。这个原则强调避免不必要的复杂性，以简洁和经济的方式解决问题：如无必要，勿增实体。</p>

<blockquote v-click>
<p>Q2 为什么长久的进化河狸没有进化成锯子？<sup>2</sup></p>
</blockquote>
<div class="mt-4 mb-4">
  <ul class="text-3">
    <li v-click>《自私的基因》：鸡只是蛋生蛋的工具，河狸的基因认为用牙齿伐木生存就足够延续自己</li>
    <li v-click>《物种起源》：生物的进化是由于环境的压力和生存的需求驱动的，如果一个生物在其生活环境中没有特定的需求，那么这个特性就不太可能出现</li>
  </ul>
</div>

<blockquote v-click>
<p>Q3 怎么才能“朝着锯子去进化”？</p>
</blockquote>
<div class="mt-4 mb-4">
  <ul class="text-3">
    <li v-click>从Q2基础上看，在没有来自环境和生存的压力和需求物种很难进化和产生新特性，那如果主动制造一些需求是不是就能朝着目标方向前进？</li>
  </ul>
</div>

# 我到底要会什么？
总结下可能在我身上发生的剧本，我不在疑虑我会什么，因为似乎不管我会什么最终都会被新技术取代。
 
<blockquote v-click>
<p>换条赛道？</p>
</blockquote>
<div class="mt-3 mb-3">
  <ul class="text-3">
    <li v-click>打不过就加入：不断学习新技术，参与新技术的研发，卷到精通18般AI技术，成功入职OpenAI，收获AI红利</li>
    <li v-click>自立门派：凭借自己或是团队的资源去制作产品，或许是夹缝中求生，或是改变世界（Fabrice Bellard）</li>
    <li v-click>走向艺术圈：AI能够创作出音乐、绘画和文学作品，但涉及深层的情感表达和个人经验，机器难以学习，成为一名Tony老师是个选择(注：美发总监)</li>
    <li v-click>成为灵魂工程师：教育行业涉及到人类的思维、创新能力以及道德价值观方面，机器非常难取代父母或老师的教育</li>
    <li v-click>决策制定：虽然AI能实现数据分析和趋势预测，但涉及到战略规划和复杂决策人类的直觉、经验和判断力AI只能作为参考，例如医生和LNG焊工</li>
  </ul>
</div>

<blockquote v-click>
<p>追逐进步，不同的学习方式</p>
</blockquote>
<div class="mt-3 mb-3">
  <ul class="text-3">
    <li v-click>意志坚定的以传统的学习模式：看书，视频，论文或是课程</li>
    <li v-click>用魔法对抗魔法，在传统模式上添加现代工具，用AI和新技术替代搜索引擎</li>
    <li v-click>漫无目的，遇到问题再出发，尝试掌握出现的原因和解决方法，也使用前两点的工具</li>
  </ul>
</div>

<blockquote v-click>
<p>总结</p>
</blockquote>
<div class="mt-3 mb-3">
  <ul class="text-3">
    <li v-click>人难以被学习的是思想和情感：即使发明了太空电梯，图恒宇还是缺少一个真实的图丫丫</li>
    <li v-click>掌握足够有难度的上游能力不容易被取代：算法核心，架构设计，芯片，GPU，核电...</li>
    <li v-click>AI不能为决策负责：人类不能惩罚机器，机器产生的错误都是由人类承担</li>
  </ul>

# 开源世界的独行侠：Fabrice Bellard
> 他发明了QEMU、FFmpeg、Tiny C编译器，还发现了计算圆周率的高效算法，用个人计算机打破了超级计算机的记录，他的名字几乎每周都被提起，但是互联网上关于它的采访，言论和营销都寥寥无几，他的个人网站：[bellard.org](https://bellard.org/)罗列了他自中学以来参与和完成的项目，横跨编译器，虚拟化，多媒体处理，数学和算法，网络技术和操作系统。

<blockquote v-click class="mt-3 mb-3">
<p>关于Bellard：“我除了是FFmpeg和QEMU等项目的开发者外，其他的我不愿多谈”</p>
</blockquote>
<div class="mt-3 mb-3">
  <ul class="text-3">
    <li v-click>1989 年，贝拉写了一个压缩器 LZEXE，成为了 DOS 环境下很受欢迎的一个压缩软件，这一年，他 17 岁。</li>
    <li v-click>1997 年，他发现了最快速的计算圆周率的算法，后被称之为 “贝拉公式”。2009年，贝拉使用个人电脑打破了圆周率计算的世界纪录，算出小数点后2.7万亿位，超过了由目前排名世界第47位的T2KOpen超级计算机于2009年8月17日创造的世界纪录。</li>
    <li v-click>1998 年，他编写了一个简洁的 OpenGL 实现 TinyGL，如果你对图形学感兴趣，可以了解这个项目。</li>
    <li v-click>2000 年，他开始构建FFmpeg。</li>
    <li v-click>2001 年，他制作了一个大小近4KB的c语言编译器子集 OTCC，也是TinyCC的前身。</li>
  </ul>
</div>
<blockquote v-click class="mt-3 mb-3">
<p>除了不可复制的天才头脑，值得我们学习的</p>
</blockquote>
<div class="mt-3 mb-3">
  <ul class="text-3">
    <li v-click>为什么研究如此多不同领域的主题：“这不是一个决定，总做同样的事情让我觉得乏味，所以我会尝试给自己换个项目”</li>
    <li v-click>在参与了多个项目后，长久的积累和知识储备让他仅仅使用170次提交就完成了QuickJs引擎的开发（对比Chromium v8)</li>
    <li v-click>我们应该只论事实，不因感受而欺骗自己，机器本身不会出错，机器的故障都是因为人的设计和维护不当导致</li>
  </ul>
</div>

# 方案/技术调研
* 电商平台上已有类似的产品，但无法扩展和接入自定义API，数据需要存储在厂商的服务器上，只能自制。
* 我需要功耗最低（最安全）的方案，最好没电也能正常显示数据：墨水屏（工作原理是悬浮在液体中的带电纳米粒子受到电场的作用而产生迁移，掉电也能显示最后一帧画面）
* 我需要支持无线WIFI的方案：ESP32/ESP8266（最终因为板载内存大小选择了ESP32）
* 我需要后续可以自己扩展模块：自建Web服务和使用Arduino UNO作为基座
<blockquote v-click class="mt-3 mb-3">
<p>学习路线</p>
</blockquote>
<div class="mt-3 mb-3">
  <ul class="text-3">
    <li v-click>U8g2不支持中文，需要对中文每个字取模存储到c文件中：U8g2_wqy_Arduino</li>
    <li v-click>网络NTP时钟同步，订阅同步cn.pool.ntp.org</li>
    <li v-click>网络接入模式：STA和AP模式，需要在联网失败的情况下将ESP32作为接入点将数据写入EEPROM</li>
    <li v-click>地理位置：根据IP和geoip库映射</li>
    <li v-click>扩展模块：光照/湿度/舵机等控制方法</li>
    <li v-click>自建Web服务：用于请求心知天气/Azure接口</li>
    <li v-click>学习Blender建模软件和3D打印：根据自己建造的模型打印成品</li>
  </ul>
</div>
